// Шардирование: по GeoHash с созданием шардов на каждую географическую зону => обеспечивает быстрый поиск публикаций по заданной стране или в радиусе от геометки
// Внутри гео шарда применяем хэш-шардирование по UserId

Table follows {
  following_user_id integer [not null]
  followed_user_id integer [not null]
  created_at timestamp
}

Table users {
  id integer [primary key]
  username varchar
  email varchar
  created_at timestamp
}

Table publications {
  id integer [primary key]
  title varchar
  Description text
  user_id integer [not null]
  status varchar
  publish_date timestamp
  latitude double
  longitude double
}

Table files {
  id integer [primary key]
  publication_id int [not null]
  s3_url varchar
}

Table comments {
  id integer [primary key]
  publication_id int [not null]
  text varchar
  user_id integer [not null]
}

Table likes
{
  user_id integer [primary key]
  publication_id integer [primary key]
  date timestamp
}
